正则表达式

 Posted on October 15, 2018  |  2 minutes (271 words)

A Regular Expression is a sequence of characters for defining a search pattern.

在线测试正则表达式

Example

php 示例

$text = 'this is an example of regular expression';
$pattern = '/\w+s\b/'; //the pattern means a word end with 's'
preg_match_all($pattern, $text, $match);
var_dump($match);
//输出
/*
array(1) {
  [0]=>
  array(2) {
    [0]=>
    string(4) "this"
    [1]=>
    string(2) "is"
  }
}
*/

1 基本构成　

元字符 描述
. 匹配除换行符以外的任意字符
\w 匹配字母、数字、下划线、汉字
\s 匹配任意空白符
\d 匹配数字
\b 匹配单词的开始或结束
^ 匹配字符串的开始
$ 匹配字符串的结束
- 表示范围
[] 匹配括号中任意一个字符
*、+、? 量词
  • 反义
元字符 描述
\W 匹配不是字母、数字、下划线、汉字
\S 匹配非空白符
\D 匹配非数字
\B 匹配不是单词的开始或结束的位置
[^aeiou] 匹配除了aeiou之外的字符

注意：反义不可多用，因其扩大了匹配范围，可能匹配到不是原来要的

  • 量词
限定符代码 描述
* 重复0次或更多次
+ 重复一次或更多次
? 重复0次或一次
{n} 重复n次
{n,} 重复n次或更多次
{n,m} 重复n到m次
  • 分支：|
    例子：
    (c|h|f|to)at : 匹配cat,hat,fat,toat
    [ch]at == (c|h)at

注意： 括号可有可无，但是正则匹配的规则是：从左到右测试每个条件，如果满足了某个分支，就不会再考虑其他分支

2 分组, 引用和断言

分组就是对正则表达式进行用()分割后命名，便于匹配内容的提取。

默认的分组命名方式是：整个正则表达式为分组0，从左到右分组为1、2、3以此类推, 也可以自己进行命名。

2.1 Example

$text = 'this is an example of regular expression';
$pattern = '/\w*(e).*\1\w*/'; //\1 表示分组1，即(e)里的字母e
preg_match_all($pattern, $text, $match);
var_dump($match);
/*
array(2) {
  [0]=>
  array(1) {
    [0]=>
    string(29) "example of regular expression"
  }
  [1]=>
  array(1) {
    [0]=>
    string(1) "e"
  }
}
*/

2.2 语法

  • 捕获
代码 描述
(exp) 匹配exp，并捕获文本到自动命名的组内
(?exp) 匹配exp，并捕获文本到name的组里面，尖括号可以换成”
(?:exp) 匹配exp，不捕获匹配的文本，也不给此分组分配组号
  • 零宽断言
代码 描述
(?=exp) 匹配exp前面的位置（顺序肯定环视, 零宽度正预测先行断言）
(?<=exp) 匹配exp后面的位置（逆序肯定环视, 零宽度正回顾后发断言）
(?!exp) 匹配后面跟的不是exp的位置（顺序否定环视, 零宽度负预测先行断言）
(?<!exp) 匹配前面不是exp的位置（逆序否定环视, 零宽度负回顾后发断言）

js默认不支持正回顾后发断言（2）和负预测先行断言（3）

  • 注释
代码 描述
(?#comment) 提供注释阅读，不对正则产生影响
  • 反向引用 用于重复搜索前面某个分组匹配的文本。如：匹配"This is a 'string'"里面的双引号里的内容"This is a 'string'"，可以这么写：(\"|').*?(\"|')，但此时匹配的却是"This is a '，这是因为|分支的特性。如果采用反向引用：(\"|').*?\1，或者利用命名捕捉组。

2.3 特性: 贪婪/懒惰匹配模式

贪婪：当正则表达式中包含能接受重复的限定符时，匹配尽可能多的字符（默认）
懒惰：当正则表达式中包含能接受重复的限定符时，匹配尽可能少的字符，只需在重复的限定符后面加上?

懒惰限定符：

懒惰限定符代码/语法 描述
*? 重复0次或更多次，但尽可能的少重复
? 重复一次或更多次，但尽可能的少重复
?? 重复0次或一次，但尽可能的少重复
{n,}? 重复n次或更多次，但尽可能的少重复
{n,m}? 重复n到m次，但尽可能的少重复

注意：实际开发中贪婪与懒惰是比较常用的，使用懒惰模式可以减少回溯，提高效率

3 正则表达式的运算符优先级

转义符 > 括号和中括号 > 限定符 > 定位点和序列（字符）>替换

4 正则表达式的模式

正则表达式的模式是用于指定匹配范围的

  • 忽略大小写（i）
  • 多行模式（m）
  • 点号通配模式（s）:点号通配换行符
  • 懒惰模式（U）:相当于前面的?
  • 结尾模式（D）:匹配时限定$匹配出来的不可有换行
  • 支持UTF-8转义表达（u）:使表达式支持UTF-8

5 正则表达式的应用

  • 校验手机、email
  • 数据安全（转义跟过滤、PHP中的strip_tags）
  • URL重写与搜索引擎优化（mysql通过REGEXP操作符来进行正则表达式的匹配）
  • 删除文件中的空行和注释（发布代码时使用）

6 正则表达式的使用注意事项

  • 使用字符组代替分支条件
  • 优先选择最左端的匹配结果
  • 标准量词是匹配优先的
  • 谨慎用点号，尽可能不用*号和+号
  • 尽量使用字符串函数处理代替, 有利于增强代码可读性
  • 合理使用括号
  • 起始、行描点优化
  • 拆分大的正则表达式
  • 慎用反义
