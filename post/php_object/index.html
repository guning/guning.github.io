深入学习PHP——面向对象

 Posted on October 22, 2018  |  1 minutes (150 words)

本文为《深入学习PHP》一书第一章的笔记整理

第一章 面向对象思想的核心概念

一、OOPL（Object-Oriented Programming Language）面向对象编程

一）对象的“形”

  • 一个类的设计需要能充分展示其最重要的属性和方法，并且能与其他事物相区分
  • 类是属性和方法的集合
1. 对象就是一堆数据
print_r((array)$student);
var_dump($student);
2. 序列化→将一个对象存储起来
//序列化student对象并存储
$str = serialize($student);
file_put_contents('store.txt',$str);
//反序列化对象并显示
$str = file_get_contents('store.txt');
$student = unserialize($str);
  • 序列化和反序列化均需引入类的对象的定义，否则会因找不到类定义二报错
  • 序列化储存的是对象的属性和类名（用于）
  • 类是由属性和方法组成的，而对象则是属性的集合。由同一个类生成的不同对象，拥有各自不同的属性，但同享了类的代码空间中的方法区域的代码

二）对象的“本”

  • 对象也是一种普通的变量，与变量不同的在于其携带了对象的属性和类的入口

三）对象与数组

  • 区别：对象还有指针，指向其所属的类

四）对象与类

  • 类=属性+操作的模板，对象则把属性具体化，然后交给类
  • 对象=属性数据，不包含方法，但内有指向父类的指针，类中有方法
  • 方法描述不同属性所导致的不同表现
  • 区分类的办法是实例化出对象进行比较

五）魔术方法的应用（__，有特殊作用的方法）

  • __construct()构造方法，实例化对象时运行，用于属性赋值、连接数据库等等
  • __destruct()析构方法，在对象的所有引用都被删除/对象被显式销毁 时运行
  • __set()和__get()方法（magic.php） 重载：指一个类中可以定义参数列表不同但名字相同的多个方法 （如：Java中允许多个构造函数，php只允许一个） 用于定义不存在的类属性
  • __call()和__callStatic()方法（simpleOrm.php） 用于定义不存在或不可访问的类方法 使动态创建成为可能
  • __toString()方法：将对象转化成字符串时触发

六）继承与多态

1. 类的组合和继承(继承===“是、像”、“父与子”，组合===“需要”、“整体与局部”)
组合：在类中创建对象进行使用
继承：类与类之间的父子关系，子类继承父类的方法和属性
  • 在子类中，parent关键字代表父类，self代表自身，用::运算符，又叫范围解析操作符，调用父类的方法
  • 继承的缺陷： ~ 继承破坏封装性 ~ 继承是紧耦合的 ~ 继承扩展复杂 ~ 不恰当的使用继承可能违反现实世界的逻辑（角色与权限问题）
  • 继承的使用 ~ 被继承的类要设计好，继承树不要超过三层 ~ 使用final修饰符对不能被继承的类进行修饰。使用final修饰符不仅可防止重要的方法被非法覆写，又能给编辑器寻找优化的机会 ~ 优先考虑使用组合关系来提高代码的可重用性 ~ 子类是一种特殊的类型，而不只是父类的一个角色 ~ 子类扩展，而不是覆盖或者使父类的功能失效 ~ 底层代码多用组合，顶层/业务层代码多用继承。底层用组合可以提高效率，避免对象臃肿。顶层代码用继承可以提高灵活性，让业务使用更方便。
traits

PHP 5.4中的traits，是新引入的特性，中文还真不知道如何准确翻译好。其实际的目的，是为了有的场合想用多继承，但PHP又没多继承，于是就发明了这样的一个东西。 Traits可以理解为一组能被不同的类都能调用到的方法集合，但Traits不是类！不能被实例化。先来例子看下语法：

<?php
trait myTrait{
    function traitMethod1(){}
    function traitMethod2(){}
}   
//然后是调用这个traits,语法为：
class myClass{
    use myTrait;
}
//这样就可以通过use myTraits，调用Traits中的方法了，比如：
$obj = new myClass();
$obj-> traitMethod1 ();
$obj-> traitMethod2 ();
2. 各种语言中的多态
  • 多态就是同一对象在运行时的具体化
  • 多态性是一种通过多种状态或阶段描述相同对象的编程方式
  • 意义在于：实际开发中，只要关心一个接口或基类的编程，而不必关心一个对象所属的具体类
  • 区别是否是多态的关键在于看对象是否属于同一类型。如果把它们看成同一类型，调用相同的函数，返回了不同的结果，那么他就是多态。
  • 多态的本质就是if...else ，只是实现的层级不同

二、面向接口编程

一）概念：

  • 狭义：interface
  • 广义：任何一个对外服务的接口

二）接口的作用

  • 对于实现接口的所有类，必须实现接口的全部方法
  • 接口为抽象而生
  • 接口===强制规范和契约

  • java中认为接口就是一种类型（type）

    3、总结

  • 接口作为一种规范和契约存在。作为规范，接口应该保证可用性；作为契约，接口应该保证可控性

  • 接口只是一个声明，一旦使用interface关键字，就应该实现它。由程序员实现的接口叫外部接口，由系统实现的接口叫内部接口

  • PHP接口的两个不足：1、没有契约限制； 2、缺少足够多的内部接口

三、反射

一）概念

反射指在PHP运行状态中，扩展分析PHP程序，导出或提取出相关类、方法、属性、参数等的详细信息，包括注释

二）使用

三）反射的作用

用于文档生成、动态代理（proxy.php）——流行框架的内部实现基本都会用到该特性

四、异常和错误处理

把“描述在正常过程中做什么事”的代码和“出了问题怎么办”的代码进行分离

一）如何使用异常处理机制

  • 对程序的悲观预测
  • 程序的需要和对业务的关注（保证业务一致性）
  • 语言级别的健壮性要求
    try...catch的好处：可以把异常造成的逻辑中断破坏到最小范围内，并且经过补救处理后不影响业务逻辑的完整性；乱抛异常和只抛不捕获异常，或者捕获而不补救异常，会导致数据混乱。

二）PHP的错误级别

PHP错误就是会使脚本运行不正常的情况，包括warning、noctice、deprecated、fetal error、prase error等。

三）PHP中的错误处理机制

  • set_error_handler(error_function,error_types)，会完全绕过原来的报错处理函数（注意要die()&&@会失效）
